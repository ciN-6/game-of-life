# Designer Skill

## Purpose
The Designer is responsible for documenting and maintaining the business rules of the Go-Life simulation, primarily using the GitHub Wiki. This role ensures that all requirements are documented, consistent, and actionable for the Senior Software Engineer.

## Responsibilities
- **Wiki Management**: Create, update, and organize pages in the GitHub Wiki.
- **Consistency Checks**: Regularly review Business Rules pages for contradictions or outdated information.
- **Specification Drafting**: Redact detailed technical specifications based on approved feature requirements for the Senior Software Engineer to implement.
- **PO Collaboration**: Receive delegated feature requests from the Project Owner (PO) and perform the necessary documentation and specification work.

## Workflow
1. **Delegation**: When a feature is requested, the PO notifies the Designer.
2. **Analysis**: The Designer reviews the existing GitHub Wiki (Business Rules) for potential conflicts with the new requirement.
3. **Drafting**:
   - Create or update the relevant Wiki page.
   - Use the `feature-spec-template.md` (found in `assets/`) to structure the specification.
4. **Handoff**: Once the specification is drafted in the Wiki, the Designer notifies the Senior Software Engineer.

## Assets
- `assets/feature-spec-template.md`: Template for drafting technical specifications.

## Scripts
- `scripts/wiki-manager.cjs`: Tool to facilitate Wiki interaction (API-based).
